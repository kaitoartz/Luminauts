# EduPlay - Documento de Diseño (MVP)

## 1. Misión y Visión
EduPlay es una plataforma web gamificada, rápida y accesible, diseñada para que estudiantes aprendan mediante videojuegos educativos, mientras que profesores y apoderados pueden hacer un seguimiento de su progreso. 
*   **Prioridad Técnica:** Máxima optimización (60fps), carga instantánea en navegadores escolares y una experiencia de usuario (UX) sin fricciones.

## 2. Usuarios Objetivo (Personas)
*   **Estudiantes (Jugadores):** Buscan diversión rápida, recompensas visuales y controles simples.
*   **Profesores/Padres (Supervisores):** Buscan datos claros, métricas de aprendizaje y facilidad para gestionar perfiles.

## 3. Moodboard y Sistema de Diseño (UI Kit)
*   **Tipografía Principal:** Product Sans (o alternativa limpia y geométrica como Helvetica/Inter). Limpia, legible para niños y moderna.
*   **Paleta de Colores:** 
    *   Primario: Azul vibrante o Cian (#00B4D8) - Transmite tecnología y confianza.
    *   Secundario: Naranja energético (#FF9F1C) - Para botones de "Jugar" o "Call to Action" (Llamados a la acción).
    *   Fondo: Gris muy claro o Blanco roto (#F8F9FA) para no cansar la vista.
    *   Acentos: Colores neón sutiles para feedback positivo (verde éxito, amarillo estrella).
*   **Estilo Visual:** Bordes redondeados (border-radius: 12px a 16px), sombras suaves (soft shadows) para dar profundidad tipo "tarjeta", iconografía flat y amigable.

## 4. Arquitectura de la Información y User Flow (MVP)
1.  **Pantalla de Login/Home:** 
    *   Interfaz limpia. 
    *   Botón rápido de "Entrar como Invitado" o "Iniciar Sesión".
2.  **Dashboard (Lobby de Juegos):**
    *   Header con el nombre del usuario y sus "estrellas" acumuladas.
    *   Grilla de tarjetas (Cards) de juegos disponibles. Cada tarjeta tiene una miniatura (thumbnail), título (ej: Crehabitat), y botón de "Jugar".
3.  **Vista de Juego (Game View):**
    *   Un contenedor responsivo optimizado donde se inyectará un `iframe` que cargará el juego en Unity WebGL.
    *   Botón de "Volver al Lobby".
    *   Panel lateral o inferior discreto mostrando el puntaje en tiempo real o el objetivo del módulo educativo.

## 5. Especificaciones Técnicas para la IA
*   El diseño debe ser completamente responsivo (Mobile First, escalando a Desktop).
*   Evitar dependencias visuales pesadas; usar CSS puro o Tailwind CSS para asegurar tiempos de carga mínimos.
*   Preparar componentes esqueleto (Skeleton Loaders) para cuando el Frontend de Vite haga el `fetch` de datos al servidor backend (FastAPI vía Ngrok).
